# 信工所招聘系统测试录屏 — 结合截图的重新分析

## 1. 视频技术信息

| 项目 | 说明 |
|------|------|
| 来源 | Chrome 录屏（`encoder: Chrome`） |
| 封装 | WebM（Matroska） |
| 编码 | VP9（Profile 0），`yuv420p`，带 `alpha_mode` 元数据 |
| 分辨率 | 3840×2160（4K），约 29.17 fps |
| 时长 | ffprobe 无总时长元数据；用 FFmpeg 8.x 抽帧推算约 **7 分 30 秒** 结束 |
| 访问环境 | 录屏中页面均为 **`http://10.28.12.15/...`**（内网，与手册一致） |

**本地截取关键帧命令（需使用 Chocolatey 等新版本 FFmpeg，勿用旧版 `python\\Scripts\\ffmpeg`）：**

```powershell
New-Item -ItemType Directory -Force -Path "docs\video_frames" | Out-Null
& "C:\ProgramData\chocolatey\bin\ffmpeg.exe" -y -i "信工所招聘系统测试录制.webm" -vf "fps=1/90" -frames:v 30 "docs\video_frames\frame_%03d.png"
```

截图已生成于：`docs/video_frames/frame_001.png`～`frame_005.png`（约每 90 秒一帧）。

## 2. 各截图对应的业务阶段（按时间顺序）

### 2.1 `frame_001`（约 0:01:30）— 部门侧：岗位发布申请

- **路由**：`/positions/xgsPositionPublishList`
- **菜单**：**岗位发布管理 → 发布申请**
- **账号上下文**：侧栏显示为 **部门测试账号**（对应流程中的部门管理员 `test-bm` 一类角色）。
- **界面要点**：
  - 列表区有 **查询 / 重置**、**+ 新增**、导出、导入。
  - 弹窗 **「新增」** 中填写岗位：**招聘部门、课题组**（均带「选择」）、咨询电话、邮箱、**岗位名称**、**招聘岗位分类**（普通/副高级以上/博士后/人才派遣）、**岗位类型**（支撑/管理/科研）、研究方向、招聘人数、学历学位、专业、工作年限等；带 `*` 的为必填。
- **与自动化关系**：E2E 需覆盖 **打开「发布申请」→ 新增 → 填表 → 保存**；后续在列表中对记录执行 **申请**（手册中的申请发布）。

### 2.2 `frame_002`（约 0:03:00）— 求职者登录

- **路由**：`/login?redirect=/XgsMyInterview/XgsInterviewList`
- **账号**：用户名 **`test-qz`**（求职者）。
- **界面要点**：账号、密码、**图形验证码**（录屏中示例为 `m346` 与右侧图片一致）、**记住我**、**登录 / 注册 / 返回首页**。
- **与自动化关系**：未关闭验证码时，Playwright 需 **测试环境关闭登录验证码**（与后端 `enableLoginCaptcha` 配置一致）或 **专用测试入口**；纯脚本识别验证码不稳定。

### 2.3 `frame_003`（约 0:04:30）— 求职者：简历/应聘表单与上传

- **界面要点**：
  - 右侧为 **多段式简历表单** 导航：基本信息、工作经历、教育经历、家庭状况、工作主要业绩、应聘岗位陈述、研究方向与专长、论文专著专利、上传其它材料等。
  - **照片上传**：说明支持 JPG/PNG/GIF，≤2MB，建议 400×500。
  - 底部有 **关闭**、**提交申请**。
  - 前景为 **资源管理器 + 图片预览**（选中 `teacherqr1.jpg`），体现录屏中 **本地上传图片** 的操作路径。
- **与自动化关系**：与现有 `resume-submit.spec.ts` 一致，需 **固定测试用图片路径**、`input[type=file]` 或项目约定选择器；注意 **大屏 4K** 下布局与视口（Playwright 默认 1366×768 可能需滚动或 `scrollIntoView`）。

### 2.4 `frame_004`（约 0:06:00）— 人事处登录

- **路由**：`/login?redirect=/dashboard/analysis`
- **账号**：用户名 **`test-rl`**（人力测试账号）。
- **验证码**：示例为 **AFba**（图形验证码）。
- **与自动化关系**：与 `frame_002` 相同，**登录与验证码策略**需与测试环境配置统一。

### 2.5 `frame_005`（约 0:07:30）— 人力：面试邀请与结果

- **路由**：`/xgsinviteToInterview/xgsinviteToInterviewList`
- **菜单**：**面试管理 → 面试邀请**（侧栏同时可见 **综合查询**）。
- **账号**：**人力测试账号**（`test-rl`）。
- **界面要点**：
  - 顶部曾出现 **「编辑成功!」** 类提示，说明有 **保存/编辑** 操作。
  - 筛选：**申请人姓名、岗位名称、岗位部门**；**查询 / 重置 / 展开**；导出、高级查询。
  - 表格列含：申请人姓名、岗位名称、岗位部门、岗位类型、**面试时间、面试地点、邀请信息**、**邀请状态**（如「待邀请」「已发送邀请」）。
  - **操作列**：**查看、邀请、撤回、通过、未通过**（对应手册中面试邀请与面试结论）。
  - 分页示例：**共 14 条**，10 条/页。
- **与自动化关系**：可验证 **邀请发出 → 状态变更 → 通过/未通过**；与求职者 **「我的面试」** 接受邀请联动。

## 4. 对自动化测试方案的补充结论

1. **Base URL**：录屏全程为 **`http://10.28.12.15`**；本地 `E2E_BASE_URL=http://localhost:3100` 仅当前端代理一致时可用，**需与 `vite` 代理的后端地址一致**。
2. **验证码**：登录页在多个角色中均出现 **图形验证码**，**E2E 与接口联调** 均建议在测试环境 **关闭登录验证码** 或注入固定测试策略（与 `LoginController.validateCaptcha` 中 `enableLoginCaptcha` 一致）。
3. **关键路由（可写入 Playwright 用例）**：
   - 发布申请：`/positions/xgsPositionPublishList`
   - 登录（带 redirect）：`/login?redirect=...`
   - 面试邀请：`/xgsinviteToInterview/xgsinviteToInterviewList`
4. **分辨率**：源片为 **4K**，若脚本在 1080p 下运行，注意 **弹窗与表格** 的可见性与滚动。
